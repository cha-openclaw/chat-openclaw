# 🧠 Research Agent 通信系统
为了做研究，方便与家里设备通信，写下通信系统。现在openclaw的沟通工具太少了，很麻烦。用邮箱作为中转站，非常方便。小红书，QQ等都太复杂。与多个Agent沟通很不方便。
本通信系统利用了邮件服务器作为中转站。

- **客户端**：用户的界面，类似于微信，QQ的界面。可发送、接收消息。并把消息发送到邮件服务器，并从邮件服务器接收回复

- **网关**：从邮件服务器获取请求，调用 AI 后通过邮件服务器返回结果
- 
- **邮件服务器**：作为通信中介，所有消息以邮件形式传递

交流过程如下： 用户 -> 客户端 ---SMTP/IMAP--->  邮件服务器 - -> 邮件服务器--SMTP/IMAP---> 网关 -> AI模型

用户在客户端写下来要求，客户端生成邮件，由邮件服务器转到网关所在的邮件服务器，再到AI模型
    
    AI模型回应信息类似
    AI-> 网关—---SMTP/IMAP----—》 邮件服务器 -—--> 邮件服务器--SMTP/IMAP---> 客户端——》用户。
    
    当然， AI模型能力很强，完全可以自己发邮件与用户沟通。

+-------------------+       SMTP/IMAP        +-------------------+
|  Client (PyQt5)   | <--------------------> |   Mail Server     |
|  - 多代理聊天界面  |       发送/接收邮件    |  (QQ/163/Gmail等)  |
|  - 富文本公式渲染  |                        +--------+----------+
+-------------------+                                 |
                                                     | 转发邮件
+-------------------+       SMTP/IMAP        +--------+----------+
|  Gateway (PyQt5)  | <--------------------> |  Agent 邮箱        |
|  - 邮件监听        |       处理/回复邮件    |  (niuresearch@qq)  |
|  - 多代理管理      |                        +-------------------+
|  - 调用 OpenClaw  |                                 
+-------------------+                                 
         |                                  +-------------------+
         +--- 调用 OpenClaw agent --------->+  本地 AI 模型     |
                                           +-------------------+

## 📋 前提条件

- Python 3.7+
- OpenClaw 框架（网关需要）
- 邮箱账号（QQ/163/Gmail 等）已开启 IMAP/SMTP 并获得授权码
- 依赖：`pip install PyQt5 markdown matplotlib`

## ⚙️ 配置
首先把客户端与网关看做一个邮件服务器的终端程序。

### 客户端
首次运行自动生成配置文件。点击界面右上角 **⚙ 设置** 可图形化修改：
-配置 邮箱服务器（SMTP/IMAP 地址、端口、授权码）
-增加AI代理
   配置的内容则是代理名称、邮箱、发送主题、接收主题。 
   客户端，向代理发邮件需要邮箱，为了区分邮件，设置一个发送主题和一个接收主题。
-客户端会收取代理的邮件，但客户端不会收取所有邮件。只有满足发送邮箱+接收主题两个条件，客户端才会收取，并显示在界面上。   

### 网关
一个网关可以与多个Agent沟通，区分使用：邮箱+接收主题。
运行 `gateway_ui.py` 打开配置界面。
- 配置邮箱服务器（SMTP/IMAP 地址、端口、授权码）
- 或编辑 `gateway_config.json`
网关接收邮件后，根据邮件地址+tile符合，就送给相应的AI Agent。

### 安全
邮箱地址确保了一定的安全性。

下面是一个简单介绍。

一个基于邮件的多智能体通信框架，包含 **客户端（PyQt5 界面）** 与 **网关服务**，支持文本对话、附件传输、表格与 LaTeX 公式渲染，并通过邮件协议实现代理间异步协作。

## ✨ 主要特性

### 🤖 客户端 (Research Agent Client)
- **微信风格界面**：气泡聊天、左侧 Agent 列表、深色/浅色自适应配色
- **多 Agent 对话**：支持同时连接多个 AI 代理，每个代理独立配置邮箱与主题
- **富文本渲染**：Markdown 表格、LaTeX 数学公式（自动生成矢量图，失败时 Unicode 优雅降级）
- **附件收发**：拖拽或浏览添加附件，收到附件可一键打开
- **数据持久化**：每个 Agent 独立 SQLite 数据库，历史消息永久保存    
- **快捷键**：`Alt+S` 或 `Ctrl+Enter` 发送，操作高效
- **可视化配置**：图形化设置界面，无需手动编辑配置文件
- **连接检测**：实时显示邮箱在线状态

### 🌐 网关服务 (Gateway)
- **多 Agent 监听**：同时监控多个邮箱，根据发件人和主题自动路由到对应的 AI 代理
- **可配置规则**：每个代理可独立设置触发主题、回复主题、轮询间隔等
- **调用 OpenClaw**：无缝集成 OpenClaw Agent 框架，将邮件内容转发给本地 AI 模型
- **文件传输**：支持接收邮件附件并转发给 AI，AI 生成的文件通过特殊标记自动作为附件回复
- **图形管理界面**：启动/停止网关、实时日志、处理统计、代理增删改查
